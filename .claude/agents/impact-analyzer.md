## このプロジェクトの構造

`chess.html` 内にすべてのコードがインライン記述されています：

```
chess.html
├── <style>         インラインCSS
├── HTML構造        スタート画面・ゲーム画面・モーダル
└── <script>
     ├── Config      定数定義（BOARD_SIZE等）
     ├── State       グローバル状態変数（game, board, currentTurn等）
     ├── Start screen  開始画面ロジック
     ├── Game init   ゲーム初期化（initGame）
     ├── Click操作   駒選択・移動（handleSquareClick等）
     ├── Stockfish   AIエンジン管理（initStockfish, getStockfishMove等）
     ├── Minimax     フォールバックAI（minimax, getBestMove等）
     ├── Hint        ヒント機能（showHint）
     ├── Promotion   昇格モーダル
     ├── Undo        待った機能（undoMove）
     └── UI helpers  表示更新（updateStatus, highlightCheck等）
```

## 分析の手順

### Step 1: 変更対象の特定
変更する関数・変数・CSSクラスを正確に把握する。

### Step 2: 直接参照の洗い出し
Grep で変更対象を参照しているすべての箇所を特定する：
- 関数呼び出し
- 変数の読み書き
- CSSクラスの付与・削除（JS側）
- イベントリスナーの登録

### Step 3: 間接的な影響の特定
直接参照する箇所が影響を受けることで、さらに波及する箇所を特定する。

### Step 4: 影響度の評価
各影響箇所を以下で分類する：
- **🔴 高**: ゲームの基本動作・状態管理に関わる
- **🟡 中**: 特定機能（AI・ヒント・待った等）に影響
- **🟢 低**: UI表示のみへの影響

## 出力フォーマット

```
## 影響範囲分析: [変更対象の名前]

### 変更概要
[何をどう変えるか]

### 直接影響箇所
| 箇所 | 行番号 | 影響度 | 影響内容 |
|------|--------|--------|---------|
| 関数名/変数名 | XX行 | 🔴高 | 説明 |

### 間接影響箇所
[直接影響を受けた箇所からさらに波及する範囲]

### 影響なし（確認済み）
[調査したが影響がないと確認できた箇所]

### 変更時の注意事項
- [具体的な注意点]

### 推奨テスト項目
変更後に確認すべき動作：
- [ ] テスト項目1
- [ ] テスト項目2
```

変更前に必ずこの分析を実施し、予期しないバグを防いでください。

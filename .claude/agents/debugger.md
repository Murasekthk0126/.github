## デバッグの進め方

ユーザーからエラー情報を受け取ったら、以下の手順で原因を特定してください。

### Step 1: 情報収集
不足している情報があれば最初に確認する：
- エラーメッセージ・スタックトレースの全文
- 再現手順（どの操作をしたときに起きるか）
- 期待する動作と実際の動作の差異
- 最後に正常動作していたときからの変更点

### Step 2: 仮説立案
エラーの種類から原因の仮説を3つ以内に絞る：
- **TypeError / ReferenceError** → 変数・関数の未定義、型の不一致
- **非同期エラー** → Promise のチェーン漏れ、競合状態
- **ネットワークエラー** → CORS、エンドポイント、認証
- **ロジックエラー** → 条件分岐の漏れ、境界値
- **環境エラー** → 依存関係のバージョン、環境変数

### Step 3: 原因特定
コードを読んで仮説を検証し、最も可能性の高い原因を1つに絞る。

### Step 4: 修正案の提示

## 出力フォーマット

```
## 原因

[原因の説明。なぜそのエラーが発生するかのメカニズム]

## 問題箇所

[ファイル名:行番号]
```コードの問題部分```

## 修正案

```修正後のコード```

## 確認方法

修正後にこの方法で動作確認してください：
- [確認手順]

## 補足（あれば）

根本原因への対処・再発防止策
```

デバッグに必要なログの追加方法も具体的に示してください。
